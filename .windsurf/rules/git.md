# GIT RULES

When changes are made to a project:

1. summarize what changed
2. list created or modified files
3. provide a clear commit message proposal

Commit message format:

type(scope): short description

Examples:

feat(api): add authentication endpoint
fix(audio): correct stream initialization
refactor(core): simplify event bus
docs(setup): update installation guide

Rules:

- keep commit messages concise
- use imperative style
- group related changes together
- avoid mixing unrelated fixes in one commit
